Normal Modes & Layer Hierarchies

That document asks: what is LoRA doing to the system as a whole? This document asks a different question: what is happening inside each adapted layer, and why are we adapting these layers and not others? The answers come from normal mode analysis and the feature hierarchy of deep neural networks.

1. The Two Decompositions

LoRA involves two independent structural choices, and they answer different questions:

Choice Question Our Config Physics Analogy
Rank $r = 8$ How many independent correction modes per layer? 8 modes Normal mode truncation
Layers top 8 of 32 Where in the network to apply the correction? Layers 25–32 Spatially localized perturbation

These are orthogonal axes of the same system. Confusing them leads to a paradox that we'll resolve in Section 4. First, let's understand each axis on its own terms.

Figure 1: LoRA's two orthogonal axes. The layer axis (horizontal) selects where to perturb — only layers 25–32 are adapted. The rank axis (vertical) controls how rich the correction is within each adapted layer — 8 independent modes, ordered by singular value magnitude. The green region is where all 3.47M trainable parameters live.

2. Within Each Layer: Normal Mode Analysis

2.1 The SVD Connection

Any matrix $\mathbf{M}$ can be decomposed via the Singular Value Decomposition:

$$\mathbf{M} = \mathbf{U\Sigma V}^\top = \sigma_1 \mathbf{u}_1\mathbf{v}_1^\top + \sigma_2 \mathbf{u}_2\mathbf{v}_2^\top + \sigma_3 \mathbf{u}_3\mathbf{v}_3^\top + \cdots$$

where $\sigma_1 \geq \sigma_2 \geq \sigma_3 \geq \cdots$ are the singular values (the “energies” of each mode), and $\mathbf{u}_i, \mathbf{v}_i$ are the corresponding basis vectors (the “mode shapes”). Each term $\sigma_i \mathbf{u}_i\mathbf{v}_i^\top$ is an independent rank-1 component — a single mode of the matrix.

This is structurally identical to a normal mode expansion of a vibrating system. In vibration analysis, any complex motion decomposes into independent modes, each with its own frequency (energy) and shape. The first few modes — the ones with the most energy — capture most of the system's behavior.

2.2 LoRA as Truncated SVD

The correction LoRA learns for each adapted weight matrix is:

$$\Delta\mathbf{W} = \mathbf{B}\mathbf{A}$$

where $\mathbf{B} \in \mathbb{R}^{d \times r}$ and $\mathbf{A} \in \mathbb{R}^{r \times d}$, with $r = 8$. This product $\mathbf{BA}$ is a matrix of rank at most $r$. Written in SVD form:

$$\Delta\mathbf{W} = \mathbf{BA} \;\approx\; \sigma_1 \mathbf{u}_1\mathbf{v}_1^\top + \sigma_2 \mathbf{u}_2\mathbf{v}_2^\top + \cdots + \sigma_8 \mathbf{u}_8\mathbf{v}_8^\top$$

LoRA doesn't compute an SVD explicitly — gradient descent finds the $\mathbf{B}$ and $\mathbf{A}$ that minimize the training loss. But the result is equivalent: the learned correction is constrained to rank 8, which means it can express at most 8 independent modes of adjustment to the original weight matrix.

Figure 2: The correction matrix $\Delta\mathbf{W}$ decomposes into independent modes ordered by singular value (energy). LoRA keeps only the 8 dominant modes — the ones that carry the most correction energy. The remaining modes are below the noise floor and contribute negligibly. This is truncated SVD, structurally identical to keeping the fundamental modes of a vibrating system.

2.3 What Each Mode Does

Think of matrix $\mathbf{A}$ as a measurement operator and matrix $\mathbf{B}$ as a reconstruction operator:

Figure 3: Inside each adapted layer, $\mathbf{A}$ compresses the $d$-dimensional input to an 8-dimensional “fingerprint” (what matters for the domain correction), and $\mathbf{B}$ reconstructs a full-dimensional correction from it. The 8-d bottleneck is the information bottleneck — the same structure as a rank-8 filter bank with 8 parallel channels.

The 8-dimensional bottleneck forces the correction to live in a low-dimensional subspace. $\mathbf{A}$ learns to extract the 8 most relevant features of the input for the task of “make this layer's output more hardware-diagnostics-appropriate.” $\mathbf{B}$ learns to turn those 8 features into a correction vector in the full output space.

2.4 Why Rank 8 Is Enough

The claim implicit in choosing $r = 8$ is:

The correction needed to each weight matrix lives in a low-dimensional subspace. The singular value spectrum of $\Delta\mathbf{W}$ decays rapidly. The first 8 singular values capture the vast majority of the correction's “energy” (Frobenius norm). The remaining modes contribute corrections so small they're below the noise floor of the training process.

Empirically, our training run confirmed this: validation loss dropped 27.5% with just 8 modes per layer (3.47M parameters out of 3.2B). If the needed correction were high-rank — if many independent modes were required — rank 8 would not capture enough and the loss wouldn't budge.

In normal mode terms: the “distance” between a general-purpose LLM and a hardware diagnostics specialist, measured within any single weight matrix, is well-approximated by 8 basis vectors. Most of the correction's energy is concentrated in a handful of modes, just as most of a vibrating structure's energy is concentrated in its fundamental frequencies.

3. Across Layers: The Feature Hierarchy

3.1 The CNN Precedent

The layer hierarchy in neural networks was first understood through convolutional networks for vision, which recapitulate the structure of biological visual processing discovered by Hubel and Wiesel in the 1960s:

Depth CNN Features Biological Vision (Hubel & Wiesel)
Layers 1–2 Edges, oriented gradients Simple cells in V1 — orientation-selective edge detectors
Layers 3–5 Textures, patterns, corners Complex cells in V1/V2 — position-invariant pattern detectors
Layers 6–10 Object parts, component shapes V4 / inferotemporal cortex — part-based representations
Final layers Object identity, scene semantics High-level recognition areas — category-selective neurons
The critical observation for transfer learning: early layers are universal; late layers are task-specific. A CNN trained on ImageNet to classify dogs learns edge detectors in layer 1 that are just as useful for classifying tumors in X-rays. But its final layers — the ones encoding “this combination of features means golden retriever” — are useless for radiology. This is why classical transfer learning in vision freezes the early layers and retrains the later ones.
Figure 4: The feature hierarchy in CNNs mirrors biological visual processing. Early layers learn universal features (edges, gradients); late layers learn task-specific abstractions (object identity). The same gradient — universal to specialized — appears in transformer LLMs, and it's why LoRA (like transfer learning) targets the later layers.

3.2 The Same Hierarchy in Transformers

Transformer LLMs exhibit the same depth gradient, despite having a completely different architecture:

Depth Transformer Function Analogue
Layers 1–8 Token embeddings, basic syntax, local attention patterns Edge detection — structural primitives
Layers 9–20 Grammar, coreference, entity tracking, semantic relationships Texture/part recognition — compositional structure
Layers 21–28 Abstract reasoning, domain knowledge, stylistic voice Object recognition — high-level meaning
Layers 29–32 Task-specific output shaping, response formatting Classification head — final decision

Research on transformer internals (probing classifiers, activation patching, causal tracing) consistently finds this pattern: syntactic information concentrates in early-to-middle layers, while semantic and task-specific information concentrates in late layers.

3.3 Why LoRA Adapts the Top Layers

Our configuration adapts layers 25–32 (the top 8 of 32). The reasoning maps directly to the feature hierarchy:

The deep layers encode the equivalent of structural fundamentals for language — syntax, grammar, token relationships — the basic modes every language task requires. These are like the edge detectors in a CNN: universal, transferable, and already optimally learned during pre-training. Perturbing them risks destabilizing the entire system for negligible gain.

The top layers encode domain-specific abstractions — the high-level features that determine what kind of response the model produces. This is where “general helpful assistant” becomes “hardware diagnostics specialist.” The correction needed to shift domain behavior lives here, not in the syntactic machinery below.

In hardware terms: you don't re-verify the entire board when you're updating application firmware. The lower levels (power distribution, clock trees, basic connectivity) are proven and stable. You test at the functional level — the top of the stack — where the change actually has effect.

4. Resolving the Apparent Paradox

Here is a question that seems paradoxical at first glance:

In harmonic analysis, the fundamental modes (lowest frequency, most energy) come “first.” In neural networks, the most basic features are also in the first layers. But we're adapting the LAST layers. And we said the rank-8 modes are like keeping the “fundamental harmonics.” So which end is fundamental?

The resolution is that “fundamental” means different things on the two axes.

4.1 Along the Layer Axis: Compositional Hierarchy

Layers are ordered by abstraction level, not by importance or energy. Layer 1 isn't the “most important” layer — it's the most basic. Each successive layer composes the representations of the previous one into higher-level abstractions:

$$\text{tokens} \;\xrightarrow{\text{layers 1–8}}\; \text{syntax} \;\xrightarrow{\text{layers 9–20}}\; \text{semantics} \;\xrightarrow{\text{layers 21–32}}\; \text{domain behavior}$$

You don't adapt the atoms when you want the organism to specialize. The atoms are fine. You adapt the high-level organizational principles — the later, more abstract layers. The first layers are “fundamental” in the sense of being foundational (everything else builds on them), but they are not where domain specialization happens.

4.2 Along the Rank Axis: Energy Hierarchy

Within each adapted layer, the 8 modes are ordered by correction energy (singular value magnitude). Mode 1 carries the most correction energy — it's the dominant direction the weight matrix needs to shift. Mode 8 carries the least. This is the classical harmonic ordering: fundamental first, overtones after.

$$\sigma_1 \;\geq\; \sigma_2 \;\geq\; \sigma_3 \;\geq\; \cdots \;\geq\; \sigma_8 \;\gg\; \sigma_9 \;\approx\; 0$$

4.3 The Two Axes Are Orthogonal

Figure 5: The resolution of the paradox. The layer axis (horizontal) is a compositional hierarchy — basic to abstract. The rank axis (vertical) is an energy hierarchy — dominant modes to least. They are orthogonal. “Fundamental” means “foundational” on the layer axis but “most energetic” on the rank axis. The green region — high-abstraction layers, energy-ordered modes — is where all of LoRA's 3.47M parameters live.

The rank-8 modes are “fundamental” in the energy sense — they capture the most correction energy — but they operate within the layers that sit at the abstract end of the compositional hierarchy. There is no contradiction: one axis answers which layers need correcting (the abstract ones); the other answers how efficiently you can represent that correction (8 modes suffice).

5. Implications for This Project

5.1 What the Training Run Confirmed

Our results (val loss $2.809 \to 2.037$ with rank 8, top 8 layers) confirm two empirical claims:

  1. The hardware diagnostics correction is low-rank. Eight modes per layer captured enough of the needed adjustment to reduce validation loss by 27.5%. The correction's singular value spectrum decays fast enough that modes $9+$ contribute negligibly.
  2. The correction is concentrated in the upper layers. Adapting 8 of 32 layers (25%) was sufficient. The lower 24 layers — the syntactic and basic semantic machinery — didn't need to change for the model to learn hardware diagnostics.

5.2 Future Tuning Strategies

The two axes suggest independent strategies for pushing further:

Strategy What It Does When to Try It
Increase rank
(e.g., $r = 16$ or $32$)		 More modes per adapted layer — captures finer structure in $\Delta\mathbf{W}$ If the model gets the general domain right but lacks nuance or precision
Adapt more layers
(e.g., 16 of 32)		 Push the perturbation deeper into semantic layers If the model needs to change how it reasons about the domain, not just what it says
More training data More signal for the existing rank and layers to extract When diminishing returns suggest the current parameters have extracted all available signal

The 600-iteration loss curve showed clear diminishing returns after iteration 200 (85% of improvement already captured). This suggests the current rank and layer count have extracted most of the available signal from 385 training examples. More data — not more parameters — is likely the next leverage point.
