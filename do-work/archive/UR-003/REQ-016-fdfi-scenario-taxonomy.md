# Define FD/FI Scenario Taxonomy

## What

Create `scripts/generate_fdfi_dataset.py` with the complete FD/FI scenario taxonomy — four entry categories mapped across all hardware domains with specific diagnostic scenarios, data types, value ranges, anomaly signatures, root causes, and connecting physics.

## Detailed Requirements

- Create `scripts/generate_fdfi_dataset.py`
- Define four entry categories (FD, FI, FD+FI, TRIAGE) with characteristics:
  - FD (Fault Detection): data + "is this normal?"
  - FI (Fault Isolation): anomaly + "what's causing it?"
  - FD+FI (Combined): raw data → detect + isolate
  - TRIAGE (Honest Triage): looks like hardware, isn't
- For each hardware domain (boundary scan, signal integrity, ICT, thermal, mixed-signal, functional test, production test, environmental), define 3-5 specific diagnostic scenarios with:
  - The type of data presented (TDR trace, Icc distribution, IDCODE sequence, etc.)
  - Realistic value ranges and units
  - What the anomaly looks like
  - What the root cause is
  - What physics connects the symptom to the cause
- For the TRIAGE category, define scenarios across domains where the symptom mimics a hardware fault but the cause is software, configuration, operator error, or measurement artifact
- Print a summary: count of scenarios per domain x category

## Context

This is Phase 1, Step 1.1 of the v2-fdfi dataset augmentation cycle. The taxonomy defined here feeds into all four entry generation steps (2.1-2.4). See `docs/v2-fdfi-SPECIFICATION.md` and `docs/v2-fdfi-BLUEPRINT.md` for full context.

The existing `scripts/generate_dataset.py` produced the encyclopedic entries. This new script is for the FD/FI diagnostic entries — structurally different (specific data, analytical responses).

## Exploration

**Existing patterns found:**
- `generate_dataset.py` uses 12 topic categories with `name`, `display_name`, `subtopics`, `physics_principles` — dicts in lists
- `qa_bank.py` contains all 252 Q&A pairs as plain Python dicts, physics-grounded with specific equations/values
- JSONL format: `{"instruction": "...", "input": "", "output": "..."}`
- `print_summary()` in generate_dataset.py (lines 370-389) uses formatted text table with separator lines

**Patterns to follow:**
- Plain dict data structures (no dataclasses)
- `PROJECT_ROOT = Path(__file__).resolve().parent.parent`
- Physics must be load-bearing (not decorative) per audit criteria
- Specific numerical values throughout (not vague ranges)

**Patterns to diverge from:**
- Existing script is combinatorial (topic × style); FD/FI should be curated scenario-by-scenario
- Existing entries are encyclopedic; FD/FI entries present specific data requiring analysis
- Response structure varies by category (FD leads with detection, FI with isolation, TRIAGE with ruling out hardware)
