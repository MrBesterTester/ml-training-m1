# Retry LoRA Training After Metal OOM Crash

## What

Re-run the LoRA fine-tuning (REQ-019) that previously crashed with a Metal memory error. All other apps have been closed to maximize available unified memory on the M1 iMac (16GB).

## Context

This is a retry of REQ-019 (Step 3.1: Configure & run LoRA training), which crashed the system with a Metal OOM error during training. The training script (`scripts/train.py`) already has `batch_size=1` and `grad_checkpoint=True`. The user has restarted the computer and closed all other apps to free memory.

## Requirements

- Review scripts/train.py for any additional memory-reduction options (e.g., --max-seq-length)
- Run training: `python scripts/train.py`
- Training completes without OOM errors
- Loss decreases over iterations
- adapters/hw-diagnostics/ exists and contains adapter weight files
- Save training log to results/training_log.txt

## Why

Previous training run hit Metal memory limits even with batch_size=1 and gradient checkpointing enabled. System had to be restarted. Now retrying with more available memory (all other apps closed).
